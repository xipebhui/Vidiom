# Next Product Requirement: Storyboard Asset and Production Workspace

更新时间：2026-07-10 CST

交接对象：架构师

## 产品目标

Vidiom 下一版应在真实模型主链路已通过验收、Storyboard shot 后端编辑基础已建立、Storyboard 准备度事实源已进入 API 与导出包的基础上，完成 LibTV 脚本/故事板链路中仍未闭合的用户生产体验：**Storyboard 资产/关系审阅与 Studio 可编辑生产台**。

最新状态判断：

- 真实模型主链路已通过验收：`gpt-5.5` agent、`gpt-5.5` Storyboard、`gpt-image-2` 项目图像和导出包在同一轮真实 smoke 中全部 completed。
- 最新开发已完成 Storyboard readiness summary、per-shot blockers、API 返回和导出包一致性。
- 更早一轮已完成 Storyboard shot 编辑、新增、删除、排序的后端基础。
- 下一版仍不切换到批量分镜图、视频、音频、导演台或自由无限画布；应先让用户能在 Studio 内处理 readiness blockers，维护资产和关系，并完成导出前确认。

目标用户在生成 Storyboard 后，应能像使用 LibTV 脚本/故事板节点一样，在进入批量分镜图或视频生成之前完成以下工作：

- 按准备度和阻塞项定位需要处理的 shots。
- 在 Studio 中完成 shot 编辑、新增、删除和排序。
- 管理角色、场景、道具资产。
- 查看和调整资产与 shots 的关系。
- 整理已有项目图像与 shots 的参考关系。
- 导出包含最新 Storyboard、资产、关系、图像关联、准备度摘要和阻塞项的项目包。

本需求只定义产品目标、体验和验收标准。请架构师判断当前架构是否支撑，并拆解后续开发任务。

## 用户价值

真实模型生成的 Storyboard 是第一版生产草稿，不是最终可批量生成的镜头清单。完成本需求后，用户应能：

- 快速从 readiness summary 进入需要处理的 shots。
- 修正模型拆解出的镜头节奏、时长、画面描述、动作重点、对白/声音和提示词。
- 补上模型漏掉的镜头，删除重复或无效镜头，并调整镜头顺序。
- 维护角色、场景、道具的一致性，减少后续每个镜头重复修改提示词。
- 判断哪些 shots 已确认、哪些需要修改、哪些 prompt 还没准备好。
- 把已有项目图像作为 shot 参考或分镜图占位，并清楚知道哪些 shots 仍缺少图像关系。
- 在导出包和后续生成语境中得到与界面一致的 Storyboard 结果。
- 为下一阶段批量分镜图、批量视频和视频合成建立可信上游。

## 功能范围

### 1. Readiness 驱动的状态筛选与定位

Storyboard readiness 已成为 API 与导出事实源。下一版应把这些事实转化为 Studio 内可操作的用户流程。

用户应能按状态快速定位需要处理的 shots：

- 全部 shots。
- 未确认。
- 需修改。
- 已确认。
- prompt 未准备。
- 有阻塞项。
- 已有关联图像。
- 尚无关联图像。

筛选结果应帮助用户完成检查，而不是只提供统计数字。每个 shot 的阻塞项应在编辑上下文中可见，让用户知道下一步要补字段、改状态、补资产关系还是整理图像参考。

### 2. Studio 内 shot 可编辑工作区

用户应能在 Storyboard 工作区完成：

- 编辑现有 shot 的关键生产字段。
- 新增缺失 shot。
- 删除重复、错误或不计划生产的 shot。
- 调整 shot 顺序。
- 保存后刷新仍可见。
- 在导出包中得到编辑后的 Storyboard。

关键生产字段至少包括：

- 镜头顺序。
- 剧情或 beat 关联。
- 场景引用。
- 角色列表。
- 道具列表。
- 画面描述。
- 动作或表演重点。
- 对白或声音。
- 建议时长。
- 画幅。
- 视觉风格。
- 图像提示词。
- 审阅状态。
- prompt 准备状态。

当用户修改会影响提示词或资产语境的内容时，应能明确看到相关 shot 需要重新确认。

### 3. 角色、场景、道具资产审阅

用户应能管理 Storyboard 中提取出的项目内资产。

至少应覆盖：

- 查看角色、场景、道具资产列表。
- 编辑资产名称。
- 编辑资产描述。
- 编辑资产参考提示词。
- 编辑一致性说明。
- 新增资产。
- 删除未使用或错误资产。
- 查看资产被哪些 shots 使用。

资产编辑应服务于镜头一致性与后续分镜图生成准备，而不是作为孤立备注。

### 4. Shot 与资产关系审阅

用户应能理解并整理 shot 与资产之间的关系：

- 查看每个 shot 使用了哪些角色、场景和道具。
- 查看每个资产关联了哪些 shots。
- 调整 shot 与资产的关联关系。
- 识别缺少关键资产的 shots。
- 在资产或关系发生变化后，判断相关 shots 是否仍然 prompt ready。

关系变化后的准备度提示应可解释，让用户知道需要重新确认哪些 shots。

### 5. 项目图像参考整理

现有项目图像生成能力已经通过 `gpt-image-2` 验收。本轮不要求批量分镜图生成，但应让已有项目图像在 Storyboard 审阅中更有用：

- 用户能查看项目图像资产。
- 用户能把项目图像关联到 shot。
- 用户能移除 shot 与项目图像的关联。
- 项目图像资产本身不会因为解除 shot 关联而消失。
- 用户能区分参考图与分镜图占位。
- 用户能理解哪些 shots 已有关联图像，哪些还没有。
- 用户能在导出包中看到这些关联关系。

## 关键用户流程

1. 用户创建项目并运行真实 `gpt-5.5` agent。
2. 用户触发真实 `gpt-5.5` Storyboard 生成。
3. Storyboard 成功后，用户进入 Storyboard 工作区查看准备度摘要。
4. 用户按未确认、需修改、prompt 未准备、有阻塞项等状态定位 shots。
5. 用户逐个修改 shot 字段，调整镜头顺序，新增缺失 shot，删除无效 shot。
6. 用户查看角色、场景、道具资产，修正描述、参考提示词和一致性说明。
7. 用户检查资产与 shots 的关系，补齐缺失关系或移除错误关系。
8. 用户把已有项目图像关联到对应 shot，作为参考或分镜图占位。
9. 用户再次查看准备度摘要，确认仍有哪些 shots 需要处理。
10. 用户导出项目包，导出结果包含编辑后的 Storyboard、资产、关系、图像关联、准备度摘要和 per-shot 阻塞项。
11. 架构师和下一轮产品任务读取文档与产品状态，能判断是否已经具备推进批量分镜图生成的上游条件。

## 体验要求

- Storyboard 编辑体验应面向镜头生产，而不是普通长文本表单。
- 用户应能快速扫描哪些 shots 需要处理，哪些已经确认。
- 状态筛选和阻塞项应直接帮助用户定位下一步操作。
- 新增、删除、排序和编辑后的结果必须可追溯、可保存、可导出。
- 资产编辑应清楚影响到哪些 shots。
- 图像关联应清楚区分参考图、分镜图占位或已有项目图像。
- 如果某个 shot 缺少关键字段，应以阻塞项形式提示用户。
- 用户不应因为一次编辑失误而失去已完成的真实模型生成结果；具体撤销、版本或历史体验由架构师评估是否纳入本轮。
- 不得出现假生成结果、占位成功结果或静默覆盖。
- 不得暴露 `HM_LLM_APIKEY`、`HM_IMG_APIKEY` 或实际密钥值。

## 产品验收标准

### Readiness 使用验收

- Storyboard 工作区显示来自当前事实源的 shot 总数、已确认数量、需修改数量、未确认数量、prompt 未准备数量和有阻塞项数量。
- 用户可以从准备度摘要进入未确认、需修改、prompt 未准备或有阻塞项的 shots。
- 用户可以按全部、未确认、需修改、已确认、prompt 未准备、有阻塞项、已有图像、无图像筛选 shots。
- 每个阻塞项都能让用户理解需要处理什么。
- 准备度只用于提示和确认，不调用真实模型，不生成媒体资产。

### Shot 工作区验收

- 用户可以在 Studio 中编辑现有 shot 的核心生产字段。
- 用户可以新增 shot，并为新增 shot 填写必要字段。
- 用户可以删除 shot，删除后 Storyboard 顺序保持可读。
- 用户可以调整 shot 顺序，顺序变化会反映在 Storyboard 展示和导出包中。
- 编辑后的 Storyboard 会被保存，并在刷新后仍然可见。
- 编辑后的 Storyboard 会进入项目导出包。
- 影响提示词或资产语境的编辑会让用户重新确认相关 shot 的准备状态。

### 资产审阅验收

- 用户可以查看角色、场景、道具资产列表。
- 用户可以编辑资产名称、描述、参考提示词和一致性说明。
- 用户可以新增资产。
- 用户可以删除未使用或错误资产。
- 用户可以查看资产关联的 shots。
- 最新资产进入 Storyboard 展示和导出包。

### 关系审阅验收

- 用户可以查看每个 shot 当前关联的角色、场景和道具。
- 用户可以查看每个资产被哪些 shots 使用。
- 用户可以调整 shot 与资产关系。
- 资产或关系变化后，相关 shot 的准备状态可被用户重新判断。
- 最新关系进入 Storyboard 展示、readiness 判断和导出包。

### 图像关联验收

- 用户可以查看当前项目图像资产。
- 用户可以把已有项目图像关联到 shot。
- 用户可以移除 shot 与项目图像的关联。
- 项目图像资产本身不会因为解除 shot 关联而消失。
- 用户可以区分参考图与分镜图占位。
- 每个 shot 能显示是否已有项目图像关联。
- 导出包包含最新 shot-image 关联摘要。

### 真实模型与回归验收

- 现有真实模型配置继续保持：语言模型 `gpt-5.5`，图像模型 `gpt-image-2`，配置变量为 `HM_BASE_URL`、`HM_LLM_APIKEY`、`HM_IMG_APIKEY`。
- Storyboard 生成、项目图像生成、readiness 事实源和导出包能力不得退化。
- 最新真实 smoke completed 记录不得被删除或改写为未验收。
- 常规自动化测试不得自动消耗真实模型额度。
- 文档不得包含 secret values。

## 非目标

本轮不做以下能力：

- 不做新模型接入或模型替换。
- 不做自动改用其他模型、备用 provider、假数据或占位结果。
- 不做批量分镜图生成。
- 不做批量视频生成。
- 不做视频剪辑或视频合成。
- 不做音频生成、提取、变速或音视频分离。
- 不做导演台 3D 构图。
- 不做自由无限画布节点创建。
- 不做跨项目资产库。
- 不做完整工作流保存与复用。
- 不做主体库、风格库、自定义风格、运镜控制或真人素材合规校验。

如架构师判断某类备用策略、撤销版本、长耗时保护或降级路径对正确性必需，必须标为“需用户确认”，不得作为默认产品方案写入后续拆解。

## 与 LibTV 的对应关系

本需求对应 LibTV 脚本/故事板能力中的“生成后检查、资产管理和批量生成前确认”阶段：

- LibTV 支持编辑 shot 信息；本需求要求 Vidiom 在 Studio 中完成镜头生产字段编辑。
- LibTV 支持调整 shot 顺序、新增 shot、删除 shot；本需求要求 Vidiom 把已有后端基础转化为用户可用流程。
- LibTV 会将角色、场景、道具资产化；本需求要求 Vidiom 提供项目内资产审阅与关系管理。
- LibTV 在批量生成前强调最终提示词合成和确认；本需求要求 Vidiom 使用 readiness 和 blockers 驱动用户完成确认。
- LibTV 后续可批量生成分镜图和视频；本需求为 Vidiom 后续进入批量媒体生成建立可信上游。

LibTV 的无限画布、批量分镜图、批量视频、视频合成、音频工具、运镜预设和导演台仍是后续差距。本轮继续优先补齐 Storyboard 资产/关系审阅与 Studio 可编辑生产台，是因为它直接连接真实模型生成结果与后续媒体生成能力。
